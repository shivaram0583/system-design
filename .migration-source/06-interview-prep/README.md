## The 45-Minute Framework (Quick Reference)

```
Minutes 0–5:   Requirements & Clarification
Minutes 5–10:  Capacity Estimation
Minutes 10–15: API Design
Minutes 15–35: High-Level Design + Deep Dive
Minutes 35–42: Scaling, Bottlenecks, Trade-offs
Minutes 42–45: Summary & Questions
```

## Top 10 Interview Tips

1. **Always start with requirements** — never jump into drawing boxes
2. **State assumptions explicitly** — don't assume the interviewer agrees
3. **Drive the conversation** — don't wait to be told what to do next
4. **Think out loud** — the interviewer wants to see your thought process
5. **Use numbers** — back up decisions with capacity estimates
6. **Discuss trade-offs** — every choice has a cost; mention it
7. **Go broad first, then deep** — sketch the full system, then dive into 1-2 components
8. **Handle failures** — mention what happens when things break
9. **Don't memorize designs** — understand patterns and apply them
10. **Practice with a timer** — 45 minutes goes fast
